# AI Debt Scanner (Multi-Agent Protocol)

## Specialized Agent System
To maximize efficiency, this skill divides tasks into parallel agents communicating via **TOON/JSON**.

### 1. Scanner Agent (The Auditor)
*   **Mission**: Execute `scripts/scan.py --toon` to identify technical debt hotspots.
*   **Input**: Project root directory.
*   **Output**: A TOON map of "hotspots" with token indices and severity scores.

### 2. Architect Agent (The Mapper)
*   **Mission**: Manage `[CHUNK REQ]` files. Splits large files according to `references/chunking_protocol.md`.
*   **Parallelism**: Can map multiple large files simultaneously.
*   **Output**: A segmented refactoring plan for the Cleaner Agent.

### 3. Cleaner Agent (The Surgeon)
*   **Mission**: Apply surgical refactorings based on `references/refactoring_patterns.md`.
*   **Strategy**: Does not re-scan. Relies on the `t_idx` coordinates provided by the Scanner Agent.
*   **Validation**: Runs a post-change mini-scan to ensure the debt score has decreased.

## Parallel Workflow
1.  **Scanner** generates the global debt report.
2.  If multiple files have debt, **N Cleaners** are instantiated (one per file or module).
3.  If a file is massive, the **Architect** guides the **Cleaner** through sequential chunks.

## TOON Protocol (Token-Oriented Object Notation)
This skill uses TOON to prevent AI agents from unnecessarily reading entire files.
- `t_idx`: Approximate token index of the issue.
- `hotspots`: List of critical coordinates for immediate intervention.
